新公告摘要

Google Cloud Next’22于10月11日至13日举行。以下是我的摘要，其中包含背景信息，有助于理解(不包括Google Docs、Meet、Chat和Voice等协作云)已发布的内容。

Google Cloud Next’22总结| Google Cloud博客

想更深入吗？以下是所有123个(！)我们本周发布的公告，都在一个地方。企业…

cloud.google.com

计算

7. TPU ，是谷歌定制开发的硬件加速器，用于加速机器学习(尤其是深度学习)。启用“云TPU API”后，您可以在“计算引擎”下创建TPU

8.A100 GPU，A100是新一代nvidia GPU，甚至比V100性能更好，据 nvidia博客和对方

10.Google Batch，主要使用案例是并行计算，例如渲染、转码、基因组学&药物发现、风险建模

建立工作关系网

11–14.私人服务连接允许私人访问(使用内部IP)Google API和服务，或另一个VPC网络中的托管服务。更多的托管服务(Confluent、Databricks、Datastax、Grafana和Neo4J)正在启用“私有服务连接”(需要消化或更多解释:私有服务通过互连连接(互连不是已经是私有访问了吗？))

15.C3虚拟机系列采用200 Gbps网络，不支持Azure VM和AWS EC2的网络带宽

16.网络功能优化器，需要消化或更多解释

17–21.CDN ( 云CDN 和媒体CDN 是为了让内容更贴近客户)，动态压缩以提高性能，支持主要视频流格式(DASH/HTT动态自适应流& HLS/HTTP直播流)，视频拼接API 以轻松将广告插入视频或直播流

22–23.谷歌云防火墙是云原生防火墙服务，现在有两层:云防火墙基础版和云防火墙标准版

24–27.谷歌云盔甲是谷歌云安全和身份解决方案的一部分，专注于保护应用程序和网站免受拒绝服务和网络攻击。自适应保护动态检测异常活动，并使用机器学习模型生成签名，行业趋势

28–32.网络情报中心是Google Cloud的网络可观察性、监控解决方案(网络情报中心集成了推荐器API需要更多解释)

混合和多云

33–35. Anthos 是管理onprem、跨云(AWS、Azure)的混合多云解决方案。现在又在支持中加入虚拟机(像 Azure arc )。车队管理功能可用于不同的使用案例，例如生产/筹备/开发车队的分离，不同团队的车队

36.边缘计算是一个新的热门话题。谷歌分布式云是谷歌云对这种需求的最新回答，计算在边缘但管理在云。

开发人员生产力

38–41.更多的原生容器应用程序开发是重点。云运行是一个无服务器容器计算环境而不用担心底层机器。现在，它与云部署集成以实现持续部署，并支持与谷歌云服务的更多集成(例如，数据存储、身份、网络

开发者安全性

43–53.软件交付盾是伞式软件供应链安全解决方案，覆盖端到端软件开发生命周期，并与不同的谷歌云服务集成。组件包括:云工作站是Google Cloud上的全托管开发环境；为云代码(IDE插件家族(如VSCode、IntelliJ、PyCharm、GoLand))提供云代码源代码保护；在云工作站中支持多个JetBrains IDE；保证OSS 提供可信开源软件(OSS)包；工件注册表使您能够存储、保护和管理您的构建工件，并且容器分析主动检测工件注册表中工件的漏洞。它还增强了现有服务的安全性，如云构建、云运行、 GKE ，并将现有的(谷歌云部署、二进制授权)谷歌云产品纳入旗下

管理和迁移工具

55.谷歌云提供大型机现代化解决方案，包括双运行，支持在客户现有大型机和谷歌云上运行工作负载。一旦对两个系统的功能性能等效感到满意，新的谷歌云环境就可以成为客户的记录系统，而现有的大型机系统可以根据需要作为备份运行；用于BigQuery和云存储的大型机连接器，帮助将数据从大型机轻松移动到BigQuery和云存储

56. Workload Manager ，在Google Cloud上提供企业系统的自动化分析，目前仅支持基于SAP的工作负载

57–58.谷歌云碳足迹，碳排放量估算两者透明以支持可持续性

AI和ML

59–61.围绕企业内容数字化的文档智能。翻译中心，这是一种自助式文档翻译服务，面向将大量文档翻译成多种不同语言的组织。它使用云翻译API；文档人工智能解决方案套件包括用于数据提取的预训练模型，文档人工智能工作台可以创建新的定制模型或升级现有模型；文档人工智能仓库帮助企业存储、组织、搜索、治理和管理文档及其提取的数据和元数据

62. Vertex AI Vision 是一款低代码解决方案，用于构建具有拖放界面的计算机视觉应用

63. OpenXLA项目帮助将领先的机器学习框架(TensorFlow、PyTorch、JAX)连接到不同的硬件后端(GPU、CPU和ML加速器)

数据分析

64–66.Lakehouse， BigQuery 支持分析非结构化和流数据，如原始文档和pdf、视频和音频，甚至呼叫中心日志，使用BigQuery熟悉的SQL接口，轻松访问Google Cloud在ML、语音识别、计算机视觉、翻译和文本处理方面的许多功能； BigLake 现在支持主要的数据格式:Apache Iceberg，Linux基金会的Delta Lake，以及即将推出的Apache胡迪；与 BigQuery和Apache Spark 之间的集成覆盖了更多的场景，而 SQL 可能不是处理的最佳位置(像 SnowPark

68. Dataplex 是Google云数据目录解决方案，提供搜索和数据发现、数据质量和数据血统

商业智能

70–75.谷歌云的商业智能家族现在整合在Looker保护伞下， Looker Studio (以前的数据工作室)，支持 Looker数据模型， Looker Studio Pro ，集成到谷歌工作空间

数据库

77.随着我们从批处理过渡到流平台，CDC是一个热门领域。事务性数据库正在增加更改流:例如，云Bigtable更改流、云扳手更改流、数据流用于访问来自MySQL、PostgreSQL、AlloyDB和Oracle数据库的流式低延迟数据

80.将Vertex AI与Spanner 集成，使得Spanner中的SQL语句能够调用Vertex AI中的机器学习模型

81–83. Firestore 改进，例如取消了对写入吞吐量和并发活动连接的限制；成本有效、可扩展、计数聚合；Firestore中用于自动删除过期文档的生存时间(TTL)

网络安全

84.Chronicle Security Operations(包含 Chronicle SIEM 、 Chronicle SOAR 和Threat Intelligence)是一个现代化的云原生套件，可以更好地帮助网络安全团队检测、调查和应对威胁。

85.机密空间在机密计算文件夹中(其他还有机密虚拟机、机密GKE 、机密数据面板)。组织可以从聚合和分析敏感数据中获得共同的价值，同时保持数据的机密性和对数据的完全控制。

87. Foreseeti Security 集成到安全指挥中心有助于在攻击者利用高风险漏洞之前实施有针对性的补救措施

89–90.零信任。 BeyondCorp Enterprise 是 Google Cloud 对零信任解决方案的实施，能够实现对应用和资源的安全访问，并提供集成的威胁和数据保护。网络安全行动小组和精选合作伙伴可以帮助引导您完成零信任之旅
