# 非结构化数据的数据库

> 原文：<https://medium.com/mlearning-ai/database-for-unstructured-data-a1cf12a52d8b?source=collection_archive---------5----------------------->

![](img/a5e86b161ef38094a07e19c271e7f3d3.png)

在过去的五十年里，关系数据库已经成为世界上一些最重要的应用程序的基础。虽然还有其他可用的数据库，如键值、文档和图形数据库，但关系数据库仍然是最受欢迎的。这样做的主要原因是 SQL 或关系数据库语言对于初学者来说很容易学习，对于专家来说实现起来很有效。

然而，随着我们进入人工智能的新时代，存储结构化数据不再足够。如今，全球 80%的数据都是非结构化数据，包括原始文本、图像、音频、视频和各种其他文件类型。这些非结构化数据包含大量的关键信息，我们无法使用当前的关系数据库系统检索这些信息。好消息是，机器学习的最新进展让我们能够以前所未有的方式理解非结构化数据，我们可以通过让这项技术变得更容易获得来利用这一机会。

# 非结构化数据的 SQL

在本文中，我将介绍 Superinsight，这是一个具有内置机器学习功能的关系数据库。Superinsight 使任何具有 SQL 背景的人都能够在非结构化数据上运行查询。

Superinsight 的核心概念如下

*   Superinsight 建立在 PostgreSQL 之上，因此它拥有 PostgreSQL 的所有功能以及机器学习能力。
*   Superinsight 自动分析和索引数据类型为[TEXT]的列中的任何数据存储。
*   原始文本数据存储为值，而基于文件的数据(如图像)存储为对实际位置的引用。

# **安装**

Superinsight 可以作为独立的机器运行，也可以作为 Kubernetes 等编排框架的一部分运行。独立版本将用于本次演示。

Docker 是安装 Superinsight 最便捷的方式。在运行该命令之前，请记住更改用户名和密码。

```
docker run \
--name superinsight-db-standalone \
-p 5432:5432 \
-e SUPERINSIGHT_USER=admin \
-e SUPERINSIGHT_PASSWORD=password \
-e ENV_IMAGE_TO_LABEL=True \
superinsight/superinsight-db-standalone:latest
```

> 在这个演示中，我们将设置标志 **ENV_IMAGE_TO_LABEL=True** ，这个选项允许我们基于文本搜索图像。

# **连接**

docker 容器运行后，您可以使用 PostgreSQL 协议和默认数据库 Superinsight 在端口 5432 上连接到 Superinsight。

```
postgres://admin:password@127.0.0.1:5432/superinsight
```

对于使用数据客户端工具的访问，我们建议使用 [DBeaver](https://dbeaver.io/) 。请参见下面的设置示例。

# **存储和搜索非结构化数据**

Superinsight 可以使用标准 SQL 存储结构化和非结构化数据。Superinsight 将只分析存储在数据类型为[TEXT]的列中的非结构化数据。在下面的例子中，我们将使用 [Flipkart 产品数据集](https://www.kaggle.com/datasets/PromptCloudHQ/flipkart-products)。

## 创建表格

标准的 CREATE TABLE 语句可用于创建表。因为我们只希望 Superinsight 将 name、image 和 description 下的列作为非结构化数据进行分析，所以我们对这些列使用数据类型[TEXT]。同样，每个表必须有一个主键；这是 Superinsight 的一个要求。

```
CREATE TABLE mldb.product (
    _id serial PRIMARY KEY,
    category varchar(500),
    brand varchar(500),
    name TEXT,
    image TEXT,
    description TEXT,
    price float8,
    url varchar(500)
);
```

## 原始文本的语义搜索

我们将使用表和预构建模型之间的**连接**语句创建一个语义搜索查询。上的关键词**指示哪一列被用作语义搜索模型的输入。 **WHERE** 关键字用于指定要比较的值。最后，我们可以使用 predictions.score 作为 **ORDER BY** 对最相关的结果进行排序。**

> 注意:在我们第一次搜索之前，Superinsight 必须将最新的语义搜索模型下载到您的数据库。根据互联网带宽，这可能需要 1-2 分钟。

```
SELECT predictions.score, mldb.product.image, mldb.product.name, mldb.product.description
FROM mldb.product
JOIN model.semantic_search
ON model.semantic_search.inputs = mldb.product.description
WHERE model.semantic_search.similar = 'Something to sit on'
ORDER BY predictions.score DESC
```

Searching (**Something to sit on)** returns (**Sofa Bed)**

我们对“可以坐的东西”的语义搜索产生了四个得分大于 0.11 的沙发床记录。Superinsight 与其他关系数据库之间最重要的区别是，结果是由理解数据上下文的机器学习模型评分的，从而产生更丰富的搜索结果。

## 图像到图像的语义搜索

图像搜索就像搜索原始文本一样。需要更改的只是 **WHERE** 子句值和 **ON** 子句条件。我们将使用图像 url 作为搜索值，根据图像相似性来查找记录。

```
SELECT predictions.score, mldb.product.image, mldb.product.name, mldb.product.description
FROM mldb.product
JOIN model.semantic_search
ON model.semantic_search.inputs = mldb.product.image
WHERE model.semantic_search.similar = 'http://img5a.flixcart.com/image/short/u/4/a/altht-3p-21-alisha-38-original-imaeh2d5vm5zbtgg.jpeg'
ORDER BY predictions.score DESC
```

## 文本到图像的语义搜索

我们还可以通过用文本描述图像的上下文来搜索图像。在下面的例子中，我们正在寻找“女装”图片。

```
SELECT predictions.score, mldb.product.image, mldb.product.name, mldb.product.description
FROM mldb.product
JOIN model.semantic_search
ON model.semantic_search.inputs = mldb.product.image
WHERE model.semantic_search.similar = 'woman dress'
ORDER BY predictions.score DESC
```
